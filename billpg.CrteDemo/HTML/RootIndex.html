Cross Request Token Exchange

This website implements each side of the exchange, in a way that you can test against your own web service.

See this project's github for a full description of the exchange.

To assist developing a Caller.

This is for your service to act as the Caller and this service will act as the Issuer. You code will make a POST request to this service and will respond by making a separate GET request back to your service with the newly issued Bearer token. There's a separate end-point that will test you have a valid Bearer token.
